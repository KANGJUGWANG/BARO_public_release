# Model Overview

BARO uses a two-stage recommendation pipeline.

## Stage 1

Stage 1 estimates future price-saving potential.

## Stage 2

Stage 2 produces the BUY or WAIT decision based on the feature row and Stage 1 output.

## UI Thresholds

- Oneway: 80%
- Roundtrip: 65%

Model pkl artifacts and private training outputs are not included in this public repository.
