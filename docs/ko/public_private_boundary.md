# BARO 공개/비공개 경계

## 공개 가능한 내용

- 프로젝트 문제 정의와 사용자 시나리오
- BUY/WAIT 추천의 의미
- Stage1/Stage2 모델 구조 설명
- 편도/왕복 모델 분리 이유
- 데이터 수집 기간과 요약 row 수
- 라벨과 평가 기준의 개념
- threshold가 BUY/WAIT 전환 기준이라는 설명
- API endpoint의 공개용 contract
- frontend 화면 흐름과 상태 표시 정책
- 최종 모델 버전 문자열과 threshold 요약

## 공개하지 않는 내용

- 운영 서버 주소와 내부 네트워크 정보
- 개인 로컬 절대 경로와 서버 내부 절대 경로
- 비공개 환경 설정 파일의 실제 내용
- 인증 키, 토큰, 서명용 비밀값, 외부 서비스 키
- 모델 바이너리 파일
- 원천 DB, dump, parquet, 대용량 csv, 실험 raw output
- 운영 배포 계정, 서비스 재시작 명령의 실제 환경값
- 내부 임시 산출물 경로

## 문서화 방식

공개 문서는 재현 가능한 설계와 판단 근거를 설명한다. 그러나 실제 운영 자산을 복제할 수 있는 정보는 제외한다.

| 구분 | 공개 방식 |
| --- | --- |
| 모델 | 구조, 버전명, threshold 요약만 공개 |
| 데이터 | 기간과 row 수 요약만 공개 |
| API | endpoint와 request/response 개념 공개 |
| 배포 | 일반적인 build/deploy 절차 공개 |
| 보안 정보 | 예시 placeholder만 제공 |

## 발표/제출용 문구

BARO 공개 저장소는 서비스 구조와 모델 적용 과정을 설명하기 위한 제출용 저장소다. 실제 운영 모델 파일, 원천 데이터, 인증 정보, 서버 세부 설정은 보안과 용량 문제로 별도 보관한다.
